# 学习要点清单-03（脚本与校验细节）

资料时间：2026-02-01

## 文档解析与转换脚本
| 脚本 | 输入 | 输出 | 行为要点 | 失败与退出 |
|---|---|---|---|---|
| skills/req-parse/scripts/parse_doc.py | --input-dir 目录 | cleaned-requirements/chunks 与 assets | Docling 优先解析；PDF 可降级 PyPDF2；DOCX 可降级 python-docx；MD/TXT 直接复制 | 输入目录不存在或无解析库时退出码 1；未发现支持文件会提示 |
| skills/testcase-generator/scripts/parse_text_protocol.py | 用例文件 | 解析结果打印 | 标题格式 ## [P1..P5][反向]；字段为测试类型、前置条件、测试步骤、预期结果；步骤分隔符支持 。；，与换行 | 严格模式下字段缺失或测试类型不合法会报错 |
| script-fix/convert_tc_format.py | test-case 目录 | 原地修改 | 将 TC_ 标题与表格字段转为 [P1..P5] 格式；默认测试类型为 功能 | 仅匹配 ## TC_ 开头的标题 |

## issues.md 校验脚本
| 脚本 | 校验点 | 规则摘要 | 退出行为 |
|---|---|---|---|
| skills/req-tester/scripts/validate_issues.py | 文档头部 | 标题必须是 需求评审问题清单；头部必须包含 需求名称 | 通过返回 0；失败返回 1 |
| 同上 | 问题标题 | 形如 ## 问题 N [分类 · 优先级]；编号连续；分类和优先级必须在枚举内 | 失败时列出分组错误 |
| 同上 | 字段完整性 | 必须包含 位置、来源、测试视角、需澄清、回答建议、产品回答 | 缺失字段视为错误 |
| 同上 | 数量规则 | 来源疑问点至少 2 条；测试视角目的至少 2 条；需澄清问题至少 2 条且以问号结尾；回答建议至少 2 条 | 不满足即报错 |
| 同上 | 产品回答 | 必须有占位符或已填写内容 | 无占位符且为空会报错 |

## plan.md 校验脚本
| 脚本 | 校验点 | 规则摘要 | Hook 行为 |
|---|---|---|---|
| skills/testcase-planner/scripts/validate_plan.py | frontmatter | 建议包含 requirement 与 status；缺少仅警告 | Hook 模式从 stdin JSON 读取 file_path |
| 同上 | 结构 | 必须有 # 测试规划；至少一个 ITEM 与 POINT | 缺失 ITEM/POINT 为 error |
| 同上 | 风险等级 | 仅允许 Critical、High、Medium、Low | 无效风险等级为 error |
| 同上 | 测试关注点 | 每个 POINT 应有关注点；过短给 warning | Hook 模式只校验 plan.md，其他文件跳过 |

## 用例校验脚本
| 脚本 | 参数 | 行为要点 | 退出行为 |
|---|---|---|---|
| skills/testcase-generator/scripts/validate.py | path | 校验单文件或目录 | 失败返回 1 |
| 同上 | --check-duplicates 或 -d | 启用重复检测 | 默认阈值 0.7 |
| 同上 | --duplicates-only | 只做重复检测 | 有重复时退出码 1 |
| 同上 | --threshold 或 -t | 重复阈值 0-1 | 权重为标题0.4、步骤0.4、预期0.2 |
| 同上 | --hook-mode | 从 stdin JSON 获取 file_path | Hook 模式跳过 plan.md、assumptions.md、all_cases.md、index.md |
| 同上 | --single | 兼容旧参数 | 与 path 二选一 |

## 用例导出脚本
| 脚本 | 输入 | 输出 | 行为要点 |
|---|---|---|---|
| skills/testcase-generator/scripts/to_excel.py | all_cases.md 或 test-case 目录 | Excel 文件 | 依赖 openpyxl；all_cases.md 走合并文件解析；目录模式从目录名推断一级分组与二级分组 |

导出字段顺序
| 字段 | 说明 |
|---|---|
| 优先级 | P1-P5 |
| 用例标题 | 标题文本 |
| 操作步骤 | 原始编号步骤 |
| 预期结果 | 原始编号预期 |
| 测试类型 | 枚举值 |
| 前置条件 | 可选 |
| 是否反向用例 | 是或否 |
| 一级分组 | ITEM 目录名 |
| 二级分组 | POINT 文件名 |
| 备注 | 可选 |

## 修复脚本
| 脚本 | 作用 | 修改方式 | 风险点 |
|---|---|---|---|
| script-fix/fix_mismatch.py | 对齐步骤与预期数量 | 原地修改 | 可能补充通用预期“正确执行” |
