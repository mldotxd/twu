## 需求解析规范
| 项目 | 要点 |
|---|---|
| 输入 | original-requirements/（PDF、DOCX、图片、纯文本） |
| 输出 | cleaned-requirements/index.md + assets/ |
| 解析流程 | 扫描目录 → 识别类型 → 解析生成 chunks/ → AI 合并为 index.md |
| 图片处理 | 解析时插入 <!-- image: xxx.png --> 占位符，合并时替换为描述 |
| 格式修复 | 修标题层级、列表、表格；重复内容保留首次出现 |
| 合并规则 | 文件间用 --- 分隔，添加来源注释 <!-- 来源：文件名.md --> |

| 图片描述要求 | 说明 |
|---|---|
| 图片类型 | 原型图、流程图、表格截图、界面截图、示意图 |
| 主要元素 | 按钮、输入框、下拉框、链接等 |
| 文字内容 | 提取全部可见文字 |
| 业务含义 | 说明图片表达的流程或规则 |

| 检查清单 | 说明 |
|---|---|
| 解析完整性 | 所有文件都已解析并合并 |
| 图片处理 | assets/ 完整，描述已替换 |
| 文档质量 | 无乱码、表格完整、层级正确 |
| 输出产物 | 已生成 index.md，逻辑连贯 |

## 需求测试规范
| 识别范围 | 说明 |
|---|---|
| 应识别 | 影响用例设计或结果判断的问题 |
| 不识别 | 纯业务背景、纯技术实现、常识问题 |

| 识别维度 | 关注点 |
|---|---|
| 前因后果 | 业务规则、权限、前置条件 |
| 边界定义 | 生效条件、范围排除、限制条件 |
| 异常场景 | 失败处理、回退、冲突 |
| 交互细节 | 文案、提示、加载、空态 |
| 数据状态 | 字段定义、状态流转、数据约束 |
| 性能兼容 | 响应时间、并发、兼容范围 |

| 问题格式字段 | 要求 | 备注 |
|---|---|---|
| 位置 | 指明页码或章节 | 必填 |
| 来源 | 引用原文并列出疑问点 | 长版包含 |
| 测试视角 | 说明为什么影响测试 | 长版包含，文内有取舍建议 |
| 需澄清 | 问题清单，以问号结尾 | 必填 |
| 回答建议 | 给产品参考答案方向 | 长版包含，文内有取舍建议 |
| 产品回答 | 留空或占位 | 简洁版保留占位符 |

| 分类 | 可选值 |
|---|---|
| 分类 | 前因后果、边界定义、异常场景、交互细节、数据状态、性能兼容 |
| 优先级 | critical、warning、info |

| 检查要点 | 说明 |
|---|---|
| 一致性 | 原则、流程、示例一致 |
| 灵活性 | 不设置硬性数量限制 |
| 可执行性 | 字段完整、问题可测试 |

## 测试规划规范与 plan.md
| 项目 | 要点 |
|---|---|
| ITEM | 业务模块或需求章节 |
| POINT | 独立操作路径（场景） |
| 划分原则 | 场景法，不把等价类当作 POINT |
| 风险等级 | Critical、High、Medium、Low |
| 测试关注点 | 具体到数值和条件 |

| plan.md 结构 | 说明 |
|---|---|
| 标题层级 | ## 为 ITEM，### 为 POINT |
| 元数据 | 可用 frontmatter 记录 requirement、generated_at、status、total_items、total_points |
| 备注区 | 用 > 风险等级 与 测试关注点 |

| 输入输出位置规则 | 说明 |
|---|---|
| 用户给文档 | plan.md 与文档同级 |
| 用户给目录 | plan.md 在 {目录}/test-case/plan.md |
| 业务背景 | 若存在 CLAUDE.md，优先读取 |

| assumptions.md | 说明 |
|---|---|
| 触发 | 规划级假设存在时生成 |
| 内容 | 业务规则、边界条件、环境依赖、优先级假设 |
| 流程 | 用户确认后修正 plan.md |

## 用例生成规范与测试用例格式
| 项目 | 要点 |
|---|---|
| 输入 | plan.md + clarified-requirements/index.md |
| 输出 | test-case/{ITEM}/{POINT}.md + all_cases.md |
| 目录组织 | 按 ITEM 分目录，POINT 为文件名 |
| 合并规则 | all_cases.md 按 ITEM、POINT 汇总 |

| 优先级 | 说明 |
|---|---|
| P1 | 主场景正向流程 |
| P2 | 其他有效场景 |
| P3 | 主要异常场景 |
| P4 | 边界条件 |
| P5 | 低频场景 |

| 测试类型枚举 | 说明 |
|---|---|
| 12 种 | 功能、兼容性、易用性、性能、稳定性、安全性、可靠性、效果（AI类、资源类）、效果（硬件器件类）、可维护性、可移植性、埋点 |

| 字段要求 | 说明 |
|---|---|
| 标题 | ## [P1..P5][反向] 验证... |
| 前置条件 | 必须具体，含实际数据 |
| 测试步骤 | 编号连续，用中文句号分隔 |
| 预期结果 | 数量与步骤一致，结果可验证 |
| 不确定点 | 用 [待确认] 标记 |

| 用例生成方法 | 说明 |
|---|---|
| 输入项分析 | 提取字段与约束 |
| 等价类与边界值 | 识别有效/无效等价类与边界点 |
| 用例数量 | 不设硬性数量，按复杂度调整 |
| 校验流程 | 单点生成后校验，整体生成后去重检查 |

| testcase-plan | 1 理解业务范围 2 识别操作路径 3 避免数据混淆 4 评估风险 5 提炼测试关注点 |
| testcase-gen | 1 输入分析 2 等价类识别 3 测试价值评估 4 覆盖判断 5 用例设计 |

## 校验与解析要点
| 规范点 | 说明 |
|---|---|
| plan.md 校验 | 必须有 ITEM 与 POINT，风险等级合法 |
| 用例格式校验 | 优先级、测试类型、字段齐全，步骤与结果数量一致 |
| 重复检测 | 标题、步骤、预期权重 0.4/0.4/0.2，阈值 0.7 |
| 解析规则 | 标题与字段用正则解析，步骤按编号提取 |

## 规格文件之间的关系
| 文档 | 作用 |
|---|---|
| SPEC_tc-plan | 测试规划流程、frontmatter、assumptions.md |
| SPEC_tc-gen | 用例生成流程、边界与等价类、校验流程 |
| INPUT_OUTPUT_testcase-plan | 输入输出位置与 CLAUDE.md 读取规则 |
| INPUT_OUTPUT_testcase-gen | 产物布局、all_cases.md 必须生成 |
| FORMAT_plan | plan.md 写法与示例 |
| FORMAT_testcase | 用例字段、枚举、校验规则 |
| COT_testcase-plan | plan 识别思路与防错点 |
| COT_testcase-gen | 用例生成推理步骤与数量判断 |
