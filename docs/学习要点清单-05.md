# 学习要点清单-05（架构与重构）

资料时间：2026-02-01

## 工作流架构目标与原则
| 目标 | 说明 |
|---|---|
| 识别不确定点 | 把需求与用例中的假设显式化 |
| 合适时机确认 | 规划级与用例级分层确认 |
| 产物可审核 | 先审假设，再审产物 |
| 用户负担可控 | 集中确认，减少打断 |

| 原则 | 说明 |
|---|---|
| 少打扰 | 同类问题集中确认 |
| 假设显式化 | 每个假设都有记录 |
| 两阶段审核 | 先确认假设，再审核结果 |
| 分层展示 | 先概览，再补细节 |

## 分阶段流程与确认点
| 阶段 | 处理内容 | 主要产物 | 用户参与点 |
|---|---|---|---|
| 阶段一 | 需求解析与检验 | cleaned-requirements、issues.md、clarified-requirements | 回答 issues.md |
| 阶段二 | 测试规划 | plan.md、assumptions.md | 确认 assumptions.md |
| 阶段三 | 用例生成 | 用例文件、uncertainties.md | 确认 uncertainties.md |
| 阶段四 | 最终审核 | 完整测试方案 | 通过或反馈修改 |

## 不确定点分层
| 层级 | 示例问题 | 产物 | 确认时机 |
|---|---|---|---|
| 规划级 | 业务规则、边界条件、环境依赖、优先级 | assumptions.md | plan.md 生成后 |
| 用例级 | 测试数据、验证方式、前置条件细节、预期精度 | uncertainties.md | 用例生成后 |

## 产物结构与用途
| 文件/目录 | 用途 | 备注 |
|---|---|---|
| test-case/plan.md | 测试规划 | 评审产物 |
| test-case/assumptions.md | 规划级假设 | 确认后可删除 |
| test-case/uncertainties.md | 用例级不确定点 | 确认后可删除 |
| test-case/cases/ | 测试用例 | 按模块组织 |
| test-case/sessions/ | 会话测试记录 | 探索性测试 |

## 交互策略与模式
| 策略 | 说明 |
|---|---|
| 集中确认 | 同类问题合并确认 |
| 提供默认值 | 每个问题附带假设值 |
| 可跳过 | 用户信任假设时可略过 |
| 可追溯 | 假设与确认结果可回溯 |

| 模式 | 流程特点 | 适用场景 |
|---|---|---|
| 快速模式 | 跳过假设确认，直接生成用例 | 简单需求或时间紧 |
| 完整模式 | 规划级与用例级都确认 | 复杂需求或高风险 |

## 不确定点数量阈值建议
| 数量 | 处理方式 |
|---|---|
| 0 个 | 直接进入最终审核 |
| 1-3 个 | 标记在用例中，审核时一并确认 |
| 大于 3 个 | 单独生成 uncertainties.md 先确认 |

## 命令与一键流程
| 命令 | 功能 | 输入 | 输出 |
|---|---|---|---|
| /req-parse | 解析原始需求 | PRD/截图/PDF | cleaned-requirements.md |
| /req-test | 检验需求 | cleaned-requirements.md | issues.md |
| /req-merge | 合并需求 | requirements + issues | clarified-requirements.md |
| /testcase-plan | 生成规划 | clarified-requirements.md | plan.md + assumptions.md |
| /testcase-gen | 生成用例 | plan.md | 用例文件 + uncertainties.md |
| /testcase-fix | 修正用例 | 用例 + 确认结果 | 修正版用例 |

| 一键流程 | 说明 |
|---|---|
| /testcase-workflow --mode=full | 完整流程 |
| /testcase-workflow --mode=quick | 快速流程 |
| /testcase-workflow --from=plan | 从规划阶段开始 |

## 重构总体目标
| 目标 | 说明 |

## 阶段 1 细化要点（req-parse）
| 项目 | 说明 |
|---|---|
| 支持格式 | pdf、docx、doc、pptx、md、txt、png、jpg、jpeg |
| 输出结构 | cleaned-requirements/index.md + chunks/ + assets/ |
| index.md 头部 | 解析时间、原始文件数、图片数 |
| chunks 结构 | 每个文件独立片段，带解析方式与时间 |
| 图片描述 | 类型、布局、元素、文字、业务含义 |

| 当前问题 | 说明 |
|---|---|
| 命名不一致 | Command 与 Skill 命名不同 |
| 缺少 frontmatter | 未设置 disable-model-invocation |
| 行数过长 | SKILL.md 超过 200 行 |
| 脚本路径 | 旧路径写死为 req-parser |

| 计划优化 | 说明 |
|---|---|
| 内容拆分 | 详细指南迁移到 references/ |
| 增加参数 | 支持 $ARGUMENTS 与 --force |
| 增加反馈 | 解析过程输出进度 |
