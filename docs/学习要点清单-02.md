# 学习要点清单-02（流程、规范、工具）

资料时间：2026-02-01

## 工作流总览
| 步骤 | 命令 | 输入 | 输出 | 备注 |
|---|---|---|---|---|
| Step 0 | 初始化 | 业务背景、原始需求 | 目录结构、CLAUDE.md | scripts/init.py 可自动生成目录 |
| Step 1 | /req-parse | original-requirements/ | cleaned-requirements/index.md | 解析文档与图片 |
| Step 2 | /req-test | cleaned-requirements/index.md | cleaned-requirements/issues.md | 结构化问题清单 |
| Step 3 | /req-merge | issues.md 已回填 | clarified-requirements/index.md | 合并澄清答案 |

| Step 5 | /testcase-gen | plan.md + 完备需求 | test-case/{ITEM}/{POINT}.md + all_cases.md | 生成用例 |

## 目录结构与文件流转
| 目录/文件 | 作用 | 主要产物 |
|---|---|---|
| original-requirements/ | 原始需求 | PDF/DOCX/图片/文本 |
| cleaned-requirements/ | 解析与清洗 | index.md、chunks/、assets/ |
| clarified-requirements/ | 澄清后的需求 | index.md、assets/ |
| test-case/ | 测试产物 | plan.md、assumptions.md、用例文件、all_cases.md |

## plan.md 规范要点
| 要素 | 规则 |
|---|---|
| 层级 | ## 为 ITEM，### 为 POINT |
| 风险等级 | Critical/High/Medium/Low |
| 测试关注点 | 必须具体，有数值和条件 |
| 元数据 | 建议使用 frontmatter 标记状态与统计 |

## 测试用例格式（Markdown Text Protocol v0.2）
| 字段 | 规则 |
|---|---|
| 标题 | ## [P1..P5][反向] 验证... |
| 测试类型 | 12种枚举之一，必须完全匹配 |
| 前置条件 | 具体数据与环境状态 |
| 测试步骤 | 1. 2. 连续编号，句号分隔 |
| 预期结果 | 数量与步骤一致，可验证 |

测试类型枚举
- 功能、兼容性、易用性、性能、稳定性、安全性、可靠性
- 效果（AI类、资源类）、效果（硬件器件类）
- 可维护性、可移植性、埋点

## 不确定点与假设处理
| 层级 | 触发 | 输出 | 处理 |
|---|---|---|---|
| 规划级 | 业务规则或边界不清 | assumptions.md | 用户确认后修正 plan.md |
| 用例级 | 数据或验证方式不清 | [待确认] 标记 | 用户确认后修正用例 |
